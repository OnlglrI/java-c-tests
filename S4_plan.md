# S4 — Клиентская часть (Tkinter)

## Краткое описание
Сессия S4 посвящена разработке десктопного GUI-клиента на Tkinter. Клиент подключается к REST API (S3), реализует авторизацию, управление данными (просмотр, создание, редактирование, удаление) и хранит JWT-токен в течение сессии.

---

## Стек технологий
| Компонент       | Технология                          |
|-----------------|-------------------------------------|
| GUI-фреймворк   | Tkinter + ttk (стандартная библиотека Python) |
| HTTP-клиент     | requests                            |
| Переменные окр. | python-dotenv (опционально)         |
| Менеджер форм   | Tkinter grid / pack                 |

## Пошаговый план реализации

### Шаг 1 — Инициализация клиентской части
- [ ] Убедиться что `requests` установлен: `pip install requests`
- [ ] Создать `client/config.py` с константой `BASE_URL = "http://localhost:8000"`
- [ ] Создать `client/state.py` — класс/модуль с глобальными переменными:
  ```python
  token: str | None = None
  current_user: dict | None = None
  ```

### Шаг 2 — API-слой (обёртки над requests)
- [ ] `auth_api.py`:
  - `register(username, email, password)` → POST `/auth/register`
  - `login(username, password)` → POST `/auth/login` → сохранить токен в `state`
  - `get_me()` → GET `/auth/me` с заголовком Authorization
- [ ] `products_api.py`:
  - `get_all()`, `get_one(id)`, `create(data)`, `update(id, data)`, `delete(id)`
- [ ] `orders_api.py`:
  - `get_all()`, `get_one(id)`, `create(data)`, `update(id, data)`, `delete(id)`
- [ ] Все функции API используют хелпер `_get_headers()` → `{"Authorization": f"Bearer {state.token}"}`
- [ ] Возвращать `(data, error)` — кортеж, где `error` = None при успехе

### Шаг 3 — Точка входа `main.py`
- [ ] Создать `tk.Tk()` root окно
- [ ] Задать заголовок, размер (например 900×600), запретить ресайз или настроить
- [ ] Запустить с `LoginView` как стартовый экран
- [ ] Реализовать механизм переключения экранов: `show_frame(frame_class)`

### Шаг 4 — Окно авторизации (`login_view.py`)
- [ ] Поля: `Entry` для username, `Entry` для password (show="*")
- [ ] Кнопка "Войти" → вызвать `auth_api.login()`, при успехе перейти в `MainView`
- [ ] Кнопка "Регистрация" → открыть `RegisterView`
- [ ] Отображать ошибки через `tk.Label` красным цветом (неверные данные, сервер недоступен)

### Шаг 5 — Окно регистрации (`register_view.py`)
- [ ] Поля: username, email, password, confirm password
- [ ] Валидация на клиенте: пароли совпадают, поля не пустые
- [ ] Кнопка "Зарегистрироваться" → вызвать `auth_api.register()`
- [ ] При успехе — показать сообщение и вернуть на `LoginView`

### Шаг 6 — Главное окно (`main_view.py`)
- [ ] Разделить окно на две части:
  - Левая панель (sidebar): кнопки навигации — "Товары", "Мои заказы", "Профиль", "Выйти"
  - Правая область (content): сменяемые экраны (Frame)
- [ ] Кнопка "Выйти": очистить `state.token`, вернуться на `LoginView`
- [ ] Отобразить имя пользователя в шапке (`state.current_user["username"]`)

### Шаг 7 — Переиспользуемый виджет таблицы (`widgets/table.py`)
- [ ] Класс `DataTable(ttk.Frame)`:
  - Принимает `columns: list[str]` и `rows: list[tuple]`
  - Внутри создаёт `ttk.Treeview` + вертикальный и горизонтальный Scrollbar
  - Метод `load(rows)` — перезаполнить данные
  - Метод `get_selected()` → выбранная строка как dict
  - Биндинг двойного клика `<Double-1>` → коллбэк `on_select`

### Шаг 8 — Экран товаров (`products_view.py`)
- [ ] Загружать список товаров при открытии экрана (вызов `products_api.get_all()`)
- [ ] Отображать в `DataTable`: ID, Название, Цена, Остаток
- [ ] Кнопки:
  - "Добавить" → открыть `ProductForm` (пустую)
  - "Редактировать" → открыть `ProductForm` (с данными выбранной строки)
  - "Удалить" → confirm dialog → `products_api.delete(id)` → обновить таблицу
  - "Обновить" → перезагрузить список
- [ ] Показывать/скрывать кнопки "Добавить"/"Удалить"/"Редактировать" в зависимости от роли

### Шаг 9 — Форма товара (`product_form.py`)
- [ ] Топлevel окно (диалог) с полями: Название, Описание, Цена, Остаток
- [ ] Режим "создание" (пустые поля) и "редактирование" (поля заполнены)
- [ ] Валидация: цена и остаток — числа, название не пустое
- [ ] Кнопки "Сохранить" и "Отмена"
- [ ] При сохранении — вызвать `create` или `update`, закрыть диалог, обновить таблицу

### Шаг 10 — Экран заказов (`orders_view.py`)
- [ ] Аналогично экрану товаров
- [ ] Таблица: ID, Товар, Количество, Сумма, Статус
- [ ] Кнопки: "Создать заказ", "Изменить статус", "Удалить"
- [ ] При создании заказа — выпадающий список товаров из `products_api.get_all()`

### Шаг 11 — Форма заказа (`order_form.py`)
- [ ] Поля: выбор товара (Combobox), количество
- [ ] Автоподсчёт суммы при изменении количества
- [ ] Кнопки "Оформить" и "Отмена"

### Шаг 12 — Обработка ошибок и UX
- [ ] Все API-вызовы в `try/except requests.exceptions.ConnectionError`
- [ ] При ошибке соединения — показать `messagebox.showerror("Ошибка", "Сервер недоступен")`
- [ ] При ошибке авторизации (401) — перенаправить на `LoginView`
- [ ] Курсор "ожидания" (`win.config(cursor="watch")`) во время HTTP-запросов
- [ ] Все кнопки блокировать во время запроса (`btn.config(state=tk.DISABLED)`)

## Ключевые технические решения

- **Переключение экранов**: использовать паттерн `frame.tkraise()` или уничтожать и создавать фреймы заново
- **Глобальный state**: модуль `state.py` с переменными — простой способ передавать токен между экранами без сложных паттернов
- **API-слой отдельно от UI**: вся логика HTTP в `api/`, никаких `requests.get()` внутри view-файлов
- **ttk.Treeview**: стандартный способ отображения табличных данных в Tkinter
- **Диалоговые окна**: `Toplevel` для форм создания/редактирования, блокировать главное окно через `grab_set()`
- **Combobox**: для выбора связанных сущностей (товар в заказе) использовать `ttk.Combobox`
