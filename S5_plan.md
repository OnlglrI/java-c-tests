# S5 — Расширение функциональности (FastAPI + Tkinter)

## Краткое описание
Сессия S5 расширяет серверную и клиентскую части: добавляются расширенные бизнес-эндпоинты (фильтрация, поиск, статистика), ролевая модель доступа (RBAC), дополнительные экраны в Tkinter (admin-панель, отчёты, статистика), улучшенная обработка ошибок и уведомления.

---

## Стек технологий
| Компонент        | Технология                                  |
|------------------|---------------------------------------------|
| Бэкенд           | FastAPI + SQLAlchemy (расширение S3)        |
| Уведомления      | FastAPI WebSocket / polling через requests  |
| Фильтрация       | Query-параметры FastAPI + SQLAlchemy filter |
| Фронтенд         | Tkinter + ttk (расширение S4)               |
| Диаграммы/отчёты | matplotlib (встраивается в Tkinter)         |

## Пошаговый план реализации

### ═══ СЕРВЕРНАЯ ЧАСТЬ ═══

### Шаг 1 — Ролевая модель доступа (RBAC)
- [ ] Убедиться, что поле `role: Enum("admin", "user")` есть в модели `User` (из S3)
- [ ] Расширить `dependencies.py`:
  - `require_role(role: str)` — фабрика зависимостей для проверки роли
  - `get_current_admin = require_role("admin")`
- [ ] Обернуть все admin-только маршруты в `Depends(get_current_admin)`
- [ ] Добавить эндпоинт `PUT /users/{id}/role` — смена роли пользователя (только admin)

### Шаг 2 — Фильтрация и поиск товаров
- [ ] `GET /products/?search=...&min_price=...&max_price=...&in_stock=true`
  - Параметр `search` — поиск по name / description (LIKE)
  - Параметры `min_price`, `max_price` — диапазон цен
  - Параметр `in_stock` — только товары с `stock > 0`
- [ ] Реализовать фильтрацию через SQLAlchemy `.filter()` цепочку
- [ ] Добавить пагинацию: `?skip=0&limit=20`

### Шаг 3 — Фильтрация заказов
- [ ] `GET /orders/?status=...&user_id=...&date_from=...&date_to=...`
  - `status` — фильтр по статусу заказа
  - `date_from` / `date_to` — диапазон дат создания
  - `user_id` — для admin: фильтр по конкретному пользователю
- [ ] Добавить пагинацию аналогично товарам

### Шаг 4 — Эндпоинты статистики (`/stats`)
- [ ] `GET /stats/summary` — общая сводка (только admin):
  - Всего пользователей, товаров, заказов
  - Сумма всех заказов
  - Количество заказов по статусам
- [ ] `GET /stats/top-products` — топ продаваемых товаров (по количеству в заказах)
- [ ] `GET /stats/orders-by-date` — количество заказов по дням/месяцам
- [ ] `GET /stats/revenue-by-period?period=daily|weekly|monthly` — выручка по периодам
- [ ] Все stats эндпоинты защищены `Depends(get_current_admin)`

### Шаг 5 — Admin-эндпоинты (`/admin`)
- [ ] `GET /admin/users` — список всех пользователей с фильтрами (поиск, роль, активность)
- [ ] `PATCH /admin/users/{id}/activate` — активировать/деактивировать аккаунт
- [ ] `GET /admin/orders` — все заказы всех пользователей с фильтрами
- [ ] `PATCH /admin/orders/{id}/status` — ручная смена статуса заказа

### Шаг 6 — WebSocket / Polling уведомлений (опционально)
- [ ] Вариант A — **Polling**: эндпоинт `GET /notifications/` возвращает список непрочитанных событий, клиент опрашивает каждые N секунд
- [ ] Вариант B — **WebSocket**: `WS /ws/notifications` — FastAPI WebSocket с broadcast
  - Сервер отправляет сообщения при создании заказов, смене статуса
  - Клиент подключается при входе, отключается при выходе
- [ ] Модель уведомления: `id`, `user_id`, `message`, `is_read`, `created_at`
- [ ] Миграция для таблицы `notifications`

## Ключевые технические решения

- **RBAC на сервере**: dependency injection FastAPI позволяет лаконично защищать маршруты
- **Фильтрация через Query params**: Pydantic модели как `Query` параметры — чистый и валидируемый способ
- **matplotlib + Tkinter**: `FigureCanvasTkAgg` позволяет встроить график прямо в фрейм
- **Polling в отдельном потоке**: `threading.Thread(daemon=True)` — не блокирует UI; обновление UI только через `win.after()` (thread-safe)
- **ttk.Notebook**: вкладки в admin-панели — удобный UX для управления разными сущностями
- **Пагинация**: `skip` / `limit` параметры — стан��арт REST API, легко добавить кнопки "Следующая страница" в таблицу
